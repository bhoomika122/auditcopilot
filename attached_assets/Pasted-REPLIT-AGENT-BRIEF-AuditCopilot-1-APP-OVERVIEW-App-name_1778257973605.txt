REPLIT AGENT BRIEF: AuditCopilot
1. APP OVERVIEW
App name: AuditCopilot
One-line description: An AI-assisted audit review tool that ingests healthcare accounts payable data, flags anomalies using forensic-accounting rules, and generates auditor-ready workpapers with plain-English risk narratives.
Core problem it solves: Senior accountants and internal auditors at healthcare distributors and hospital systems spend 60–70% of their AP review time on manual scanning for ghost vendors, duplicate payments, FX misallocations, and split-PO patterns. AuditCopilot automates the detection layer (forensic rules) and the documentation layer (AI-drafted memos), so reviewers can focus on judgment-heavy decisions.
Target user: Senior Accountants, Internal Auditors, External Audit Associates/Seniors, and Controllers at healthcare companies (pharma distributors, hospital systems, medical device firms). Secondary: AI-curious finance professionals building portfolios.
2. FULL FEATURE LIST
Core features (must build):

CSV upload for AP transaction data (vendor name, invoice number, amount, date, currency, GL account, approver)
Pre-loaded synthetic healthcare AP dataset (200+ rows with planted anomalies) so the app demos without any upload
Forensic rule engine (pure Python, no AI) that flags:

Duplicate vendors via fuzzy matching (e.g., "MedSupply Inc" vs "Med Supply Inc.")
Benford's Law violations on invoice amounts (chi-square test on first-digit distribution)
Round-dollar payments (amounts ending in .00 above a threshold)
Weekend/holiday postings
Split-PO patterns (multiple invoices from same vendor just under approval thresholds like $9,999 when threshold is $10K)
FX misallocation flags (transactions where currency code doesn't match expected GL account currency)
Duplicate invoice numbers across vendors


Risk dashboard: total transactions reviewed, count of flags by category, dollar value at risk, top 10 highest-risk transactions
Toggle: "Generate AI Audit Memo with Claude" — when enabled, sends each flagged transaction to the Anthropic API and returns a 3-sentence audit memo citing the relevant standard (AU-C 240 fraud, AU-C 315 risk assessment, IAS 21 FX) and a recommended testing procedure
Export workpaper as Excel (.xlsx) with three tabs: Summary, Flagged Transactions, AI Memos
Filter and sort flagged transactions by risk category, dollar amount, or vendor

3. TECH STACK

Language: Python 3.11
Web framework: Streamlit (single-file, fast to build, looks polished out of the box, perfect for data apps)
Data: pandas, numpy, scipy (for chi-square Benford test), rapidfuzz (for fuzzy vendor matching), openpyxl (for Excel export)
AI: anthropic Python SDK (model: claude-sonnet-4-5)
Charts: plotly (interactive, looks better than matplotlib by default)
Deployment: Replit's built-in deployment (Streamlit on port 8501, configured in .replit)
Secrets: ANTHROPIC_API_KEY stored in Replit Secrets, accessed via os.environ

Do NOT use: React, Next.js, Flask, FastAPI, databases, or authentication. Streamlit only.
